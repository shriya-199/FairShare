## Anomaly Log
`expenses_export.csv` is not currently present in the workspace, so the actual anomaly log cannot be completed yet.

Implemented anomaly categories and policies:

| Anomaly type | Severity | Approval required | Handling policy |
| --- | --- | --- | --- |
| `MISSING_REQUIRED_FIELD` | error | yes | Block row until required data is fixed or row is ignored. |
| `INVALID_DATE` | error | yes | Block row; no date guessing. |
| `INVALID_AMOUNT_FORMAT` | error | yes | Block row; amount must be parseable as currency. |
| `NEGATIVE_AMOUNT` | error | yes | Block row as expense; user may ignore or later classify as settlement/refund. |
| `ZERO_AMOUNT` | error | yes | Block row because it does not affect balances. |
| `UNKNOWN_PAYER` | error | yes | Block row until payer is mapped to a known member or row is ignored. |
| `UNKNOWN_PARTICIPANT` | error | yes | Block row until participant is mapped to a known member or row is ignored. |
| `EXPENSE_DATE_OUTSIDE_MEMBER_ACTIVE_PERIOD` | error | yes | Block row because payer was not active on the expense date. |
| `DUPLICATE_EXPENSE_ROW` | warning | no | Surface possible duplicate; row may import unless user ignores it. |
| `SAME_EXPENSE_DIFFERENT_AMOUNT` | error | yes | Block row until user chooses policy/action. |
| `SETTLEMENT_LOGGED_AS_EXPENSE` | error | yes | Block row from expense import; should be skipped or handled by settlement workflow. |
| `UNSUPPORTED_OR_INCONSISTENT_SPLIT_TYPE` | error | yes | Block row until split type is corrected to equal, unequal, percentage, or shares. |
| `CURRENCY_MISMATCH` | error | yes | Block row if currency is not INR; do not treat USD or other currencies as INR. |
| `SPLIT_TOTAL_NOT_MATCHING_EXPENSE_TOTAL` | error | yes | Block row until split details are corrected or row is ignored. |
| `MEMBER_JOINED_LATER_INCLUDED_IN_OLDER_EXPENSE` | error | yes | Block row because participant was not active yet. |
| `MEMBER_LEFT_EARLIER_INCLUDED_IN_LATER_EXPENSE` | error | yes | Block row because participant had already left. |
| `BLANK_NOTES_OR_CATEGORY_AMBIGUITY` | warning | no | Surface context ambiguity; row may import. |
| `MIXED_DATE_FORMATS` | warning | no | Surface mixed formats in preview; parsed dates must be reviewed. |

For each detected anomaly, the app must record:
- Import ID.
- CSV row number.
- Original row data.
- Field or fields involved.
- Anomaly type.
- Severity: `error`, `warning`, or `info`.
- Human-readable explanation.
- Suggested action.
- Final action taken by the user or importer.
- Whether user approval was required.
- Whether the row was imported, skipped, blocked, or still requires review.

## Anomaly Policy Draft
- Do not crash on malformed rows.
- Do not silently guess.
- Rows with unknown people are quarantined until resolved.
- Rows with invalid dates are quarantined.
- Rows with unsupported/missing currency are quarantined until a currency policy is confirmed.
- Rows with split totals that do not match expense totals are quarantined.
- Rows involving inactive members are flagged; import policy depends on whether the row is household, trip, or settlement.
- Duplicate rows are flagged and skipped or quarantined according to deterministic duplicate policy.
- Every policy decision must appear in the import report.

## Database Schema
- `User`: app users and flatmates.
- `Group`: household/trip group.
- `GroupMembership`: group membership with `joinedAt`, `leftAt`, and `membershipType` so Meera, Sam, and Dev can be validated by date.
- `Expense`: normalized expense. Money is stored as integer minor units with `amountCents`, `originalAmountMinor`, `originalCurrency`, and `normalizedAmountInrCents`.
- `ExpenseSplit`: normalized split rows. Split owed values are integer cents with original and normalized INR audit fields.
- `Settlement`: recorded payments with original currency and normalized INR amount fields.
- `ExpenseMessage`: existing expense chat.
- `CsvImportBatch`: file name, original CSV text, status, summary counts, report JSON, creator, timestamps.
- `CsvRawRow`: row number, raw payload, raw text, parsed fields, status, final action, linked expense.
- `ImportAnomaly`: row number, original row data, anomaly type, severity, explanation, suggested action, final action, approval flag.
- `ImportDecision`: append-only record of reviewer actions for anomaly rows.
- `CurrencyRate`: exchange-rate records with source currency, target INR, rate in micros, effective date, and optional source.
- `AuditLog`: optional audit table for important import/schema/user actions.
- Relational migrations:
  - `20260614010000_import_system` adds import staging and anomaly persistence.
  - `20260614020000_explainable_import_schema` renames pivot tables, adds decisions, rates, audit logs, and original/normalized money fields.

## Import UI
- Route: `/imports`.
- User selects a group and uploads a CSV file.
- Browser reads the selected file text exactly and sends it to the backend.
- Backend stores the original CSV text and raw row data.
- Preview shows row data, anomalies, severity, suggested action, and action selector.
- Finalize imports rows without unresolved error anomalies.
- Generated report summary is stored on `CsvImportBatch.report` and shown after finalize.
- Anomaly decisions are stored in `ImportDecision`.

## Balance UI
- Route: `/groups/:groupId/balances/:userId`.
- Shows selected user's net normalized INR balance.
- Lists every expense or settlement affecting that user's balance.
- Shows CSV row number and raw CSV row for imported expenses.
- Route: `/groups/:groupId/recommendations`.
- Shows simplified payer/payee recommendations from normalized pairwise balances.
